# Contributing to searXena 🚀

Thank you for considering contributing! To maintain code quality and project integrity, please follow these guidelines.

## Pull Request Protocol

1.  **Fork the Repository**: Create your own fork and work on a logic-named branch.
2.  **Ensure CI Passes**: GitHub Actions will automatically check your code for syntax errors and basic linting. PRs with failing checks will not be merged.
3.  **No Personal Data**: Ensure your contributions do not contain API keys, personal emails, or sensitive structure information.
4.  **License Consistency**: By contributing, you agree that your code will be licensed under **AGPL-3.0**.
5.  **Describe Changes**: Your PR description must clearly state:
    *   What problem does it solve?
    *   What was modified?
    *   How was it tested?

## Code Standards

- Follow PEP8 for Python code.
- Ensure all new engines follow the existing structure in `core/engines/`.
- Maintain the privacy-first philosophy: No tracking, no external logs.

## Questions?
Contact us at **neuralarchlabs@gmail.com** or open a non-security issue.
